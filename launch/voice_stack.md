# XRAI Voice Stack — canonical tone reference

**Purpose:** keep every XRAI artifact (landing page, manifesto, launch posts, weekly updates, README) tuned to the same voice. When in doubt, come back here.

**The voice is NOT:** measured corporate, academic-only, spec-wonk, techbro hype.
**The voice IS:** visionary + grounded, counterculture + rigorous, poetic + honest about gaps.

---

## The primary problem (what XRAI answers)

**We are blind.** We are island minds. Code is opaque. AI is black-boxed. The web is flat. Search is broken. Education is broken. Communication is dated. There is a societal malaise. Social media brainwashes us and makes us sick. PCs, phones, and GUIs cripple our minds and bodies. Profit and algorithms drive conflict and instant gratification. People feel small and powerless. Civilization is at risk. Apathy and isolation are rampant.

This framing is the load-bearing wall. Every XRAI artifact should gesture at it, even if only by implication. If the pitch is only "a better file format," we've failed.

## Tone registers (three valid modes — pick one per artifact)

### A) **Visionary / inspirational** — landing page hero, launch tweet, demo voiceover
- Short sentences. Punchy. Alliteration OK.
- Big claims followed by concrete proof (or a concrete gap).
- No hedging. No "we believe."
- Example: *"God's-eye view. For everyone. Spoken into being."*

### B) **Manifesto / intellectual** — MANIFESTO.md, long blog, conference talk
- Lineage threads (Bush, Nelson, Berners-Lee, Hofstadter, Huxley, Kurzweil).
- Historical pattern → current rupture → forward bet.
- Honest about what died and why. Not afraid to name the malaise.
- Example: *"Every generation tried. Each hit the authoring wall. LLMs just dropped it."*

### C) **Technical / engineer** — SPEC.md, CONTRIBUTING, release notes, RFCs
- Precise. Unornamented. Citations.
- No marketing voice. Assume the reader can smell it.
- Gaps called out explicitly. "Out of scope" labeled.
- Example: *"v1.0 relations are binary. Hyperedges ship in v1.1."*

**Mix is fatal.** A tweet in voice A + a manifesto in voice B + a spec in voice C all referencing each other is fine. A single artifact that slides between voices reads as unsure.

## Banned phrases (across all registers)

- "Revolutionary"
- "Game-changing"
- "The future of X"
- "We believe" (state it instead)
- "Cutting-edge"
- "State-of-the-art"
- "Disrupting"
- "Paradigm shift"
- "Next-generation" (except literal next-gen hardware)
- "AI-powered" (redundant)
- Em-dashes used as colon substitutes ← use a colon if you mean a colon
- Any variant of "excited to announce"

If you catch yourself writing one, go back and say what you actually mean.

## Proof obligations (every artifact in voice A or B)

- Every big claim must either (a) cite a shipping artifact, (b) admit it's a gap + date the gap, or (c) mark it as a bet and accept it may fail.
- No empty superlatives. "Fastest" → cite the number. "Best" → cite the comparison. "Most open" → cite the competitor that's less open.
- If the proof doesn't exist yet, write the affirmative ambition and then flag the missing proof in a follow-up line. *"Infinite zoom across minds. Today: 1 human + 1 agent in one scene. v1.2 target: n."*

## When to use the problem framing directly

**Do use it:** MANIFESTO opening, long blog posts, "why now" moments in talks, demo voiceovers, launch-post post-dunk reveal.

**Don't use it:** SPEC.md (it's a technical doc), CONTRIBUTING.md (practical), CHANGELOG (historical), release notes (surgical).
